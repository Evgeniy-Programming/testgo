# Go-gRPC-PubSub

Легковесный, высокопроизводительный **in-memory брокер сообщений**, реализованный на **Go** с использованием **gRPC Streaming**.

Проект демонстрирует реализацию паттерна Pub/Sub (Издатель-Подписчик) для микросервисной архитектуры, обеспечивая доставку событий в реальном времени с минимальными задержками. Ориентирован на использование в cloud-native средах (Docker/Kubernetes).

### Архитектура и Дизайн

Проект следует структуре **Standard Go Project Layout** и принципам **Clean Architecture**.

*   **`cmd/server`**: Точка входа, DI-контейнер, инициализация конфигурации.
*   **`internal/subpub`**: Ядро бизнес-логики. Чистая Go реализация, изолированная от транспортного слоя.
*   **`internal/server`**: Транспортный слой (gRPC), зависящий от ядра через интерфейсы.

**Ключевые технические решения:**
*   **Concurrency Control**: Использование `sync.RWMutex` вместо обычного `Mutex`. Это оптимизирует производительность для сценариев `Read-Heavy` (когда чтений/рассылок значительно больше, чем подписок/отписок).
*   **gRPC Streaming**: Использование Server-Side Streaming позволяет пушить события клиентам мгновенно, избегая накладных расходов на polling.
*   **Graceful Shutdown**: Обработка системных сигналов (`SIGINT`, `SIGTERM`) и `context.Context` для корректного завершения активных соединений и горутин перед остановкой пода (важно для K8s).

### Стек технологий:

*   **Language**: Go 1.21+ 
*   **Transport**: gRPC / Protobuf
*   **Configuration**: YAML (Clean config management)
*   **Testing**: Native `testing` package + Race Detector.
